MailWhere 시작하기
=============================

1. 이 폴더의 MailWhere.exe 를 실행하세요.
2. 처음에는 [진단 실행]을 누릅니다.
3. 관리형 PC에서는 smoke gate 전까지 자동 watcher를 켜지 마세요.
4. [최근 1개월 스캔]을 누르면 Outlook Classic Inbox에서 최근 30일 메일을 갯수 제한 없이 읽고 할 일/회의/마감 후보를 로컬 DB에 저장합니다.
5. 스캔 중에는 진행 상태와 [스캔 중지] 버튼이 표시됩니다. LLM timeout이 나도 전체 스캔을 죽이지 않고, 초기/대량 스캔에서는 후보별 팝업을 띄우지 않고 완료 요약 1회만 MailWhere 토스트로 알립니다.
6. 오늘의 업무 보드는 기본 08:00에 한 번 뜹니다. 그 이후 앱/PC가 켜졌다면 다음 정시에 한 번 표시됩니다.
7. 업무 보드는 상단 [업무 보드] 버튼, tray 우클릭 메뉴, 알림 클릭에서 다시 열 수 있습니다.
8. 업무 보드에는 [오늘], [7일 내], [30일 내], [기한 미정] 필터가 있고, 왼쪽은 할 일/오른쪽은 일정입니다.
9. 각 카드의 [열기]는 Outlook 원본 메일을 열고, [삭제]는 MailWhere 업무보드에서만 숨깁니다. 원본 메일은 삭제하지 않습니다.
10. 기한이 없는 항목은 [기한 설정]을 눌러 캘린더에서 날짜를 지정할 수 있습니다.
11. LLM은 기본 OFF입니다. Ollama native 또는 OpenAI-compatible local endpoint를 설정한 뒤 [모델 불러오기] → [연결 테스트] 순서로 확인하세요.
12. LLM 실패 시 기본값은 검토함에 남김입니다. 규칙 기반 fallback은 앱이 물어볼 때 동의하거나 고급 설정에서 직접 바꾼 경우에만 사용합니다.
13. 팀 공통 endpoint/model을 배포하려면 `MailWhere.defaults.sample.json`을 `MailWhere.defaults.json`으로 복사해 수정한 뒤 exe와 같은 폴더에 두세요. 첫 실행 때 사용자 설정으로 복사됩니다.

포함 파일
- MailWhere.exe: 실행 파일
- appsettings.sample.json: 설정 예시
- MailWhere.defaults.sample.json: 팀/부서 기본 설정 seed 예시
- docs/: 운영/보안/검증 문서
- BUILD-MANIFEST.json: 빌드 정보

안전 경계
- 메일 발송/삭제/이동/답장/첨부 자동 분석을 하지 않습니다.
- vendor-specific mailbox export files 파일을 직접 파싱하지 않습니다.
- raw mail body와 prompt 로그를 저장하지 않습니다.
