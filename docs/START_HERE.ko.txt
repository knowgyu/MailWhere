MailWhere 시작하기
=============================

1. 이 폴더의 MailWhere.exe 를 실행하세요. 앱은 기본적으로 tray에 조용히 상주합니다.
2. tray 메뉴의 [열기]로 메인 창을 열 수 있고, [오늘 업무 보기]로 업무 보드만 바로 열 수 있습니다.
3. 처음에는 [진단 실행]을 눌러 Outlook COM, 저장소, 알림, LLM endpoint 상태를 확인합니다.
4. 관리형 PC에서는 수동 메일 확인이 성공하고 운영 정책상 문제가 없을 때까지 [새 메일 자동 확인]을 켜지 마세요.
5. [지금 메일 확인]을 누르면 Outlook Classic Inbox/Sent Items에서 최근 메일을 read-only로 읽고 할 일/일정/회신 대기 후보를 로컬 DB에 저장합니다.
6. 확인 중에는 진행 상태와 [확인 중지] 버튼이 표시됩니다. LLM timeout이 나도 전체 확인을 죽이지 않고, 초기/대량 확인에서는 후보별 팝업 대신 요약 알림과 검토 후보/보드 중심으로 처리합니다.
7. 기본 08:00 오늘 업무 보드는 직접 열리거나 기존 보드를 오늘 보기로 갱신합니다. 보드 열기에 실패한 경우에만 오늘 브리핑 알림으로 대신 안내합니다.
8. 업무 보드는 [전체], [오늘], [7일 내], [30일 내], [기한 미정] 필터가 있고, 왼쪽은 내가 할 일/오른쪽은 기다리는 중입니다.
9. 각 카드의 [열기]는 가능한 경우 Outlook 원본 메일을 엽니다. [나중에]는 지정 시간까지 다시 표시하지 않고, [보관]은 active 목록에서 제외해 다시 자동 표시하지 않습니다. Outlook 원본 메일은 삭제하지 않습니다.
10. [수정] 또는 카드 더블클릭으로 AI가 만든 제목, 간단한 유형, 기한을 바로잡을 수 있습니다.
11. 기한이 없는 항목은 카드의 [기한 추가] 또는 [기한 변경]에서 날짜를 지정할 수 있습니다.
12. LLM은 기본 OFF입니다. Ollama native 또는 OpenAI-compatible local endpoint를 설정한 뒤 [모델 불러오기] → [연결 테스트] 순서로 확인하세요.
13. LLM 실패 시 기본값은 검토 후보에 남김입니다. 규칙 기반 fallback은 앱이 물어볼 때 동의하거나 고급 설정에서 직접 바꾼 경우에만 사용합니다.
14. 팀 공통 endpoint/model을 배포하려면 MailWhere.defaults.sample.json을 MailWhere.defaults.json으로 복사해 수정한 뒤 exe와 같은 폴더에 두세요. 첫 실행 때 사용자 설정으로 복사됩니다.

포함 파일
- MailWhere.exe: 실행 파일
- appsettings.sample.json: 설정 예시
- MailWhere.defaults.sample.json: 팀/부서 기본 설정 seed 예시
- docs/: 운영/보안/검증 문서
- BUILD-MANIFEST.json: 빌드 정보

안전 경계
- 메일 발송/삭제/이동/답장/읽음 처리/첨부 자동 분석을 하지 않습니다.
- vendor-specific mailbox export 파일을 직접 파싱하지 않습니다.
- raw mail body와 prompt 로그를 저장하지 않습니다.
- MailWhere의 나중에/수정/보관은 로컬 업무 상태만 바꿉니다.
