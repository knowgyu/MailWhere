# Autoresearch Mission: MailWhere + OfficeWhere Codex CLI Integration

## User intent
Analyze the local `MailWhere` and `OfficeWhere` repositories and determine how to combine them into a powerful Codex CLI workflow, not as two standalone apps. Consider creating Codex/OMX skills, hooks, agents, or related CLI surfaces.

## Success criteria
- Ground findings in local files from both repositories.
- Identify each project's capabilities, tech stack, entry points, data/API surfaces, and constraints.
- Propose concrete Codex CLI integration patterns that use both projects together.
- Recommend a practical MVP and follow-on roadmap.
- Produce a report artifact suitable for implementation planning.

## Validation regime
Use `prompt-architect-artifact`: completion requires an architect-style approval verdict in `result.json` and a pointer to the output report.
